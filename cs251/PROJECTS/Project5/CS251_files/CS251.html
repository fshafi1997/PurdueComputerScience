Overview

The input for this project is a connected undirected graph. Create an adjacency lists representation of the graph, where every adjacency list's contents are sorted by increasing order (this sortedness requirement is for the purpose of obtaining the output we expect, so that we can auto-grade it). Then do a depth-first search of that graph starting from its highest-numbered vertex, during which you must create a directed version of the input graph by assigning a direction to each of its undirected edges. For tree edges the assigned direction is from parent to child, for non-tree edges the assigned direction is from descendent to ancestor. Finally, determine whether the created directed graph is strongly connected or not. If the directed graph is not strongly connected, compute the smallest number of directed edges that would have to be added to the directed graph, in order to make it strongly connected.

Implementation

Create a class StronglyConnected.java to perform the task described above. Your program should process input using StdIn.java in the format described on slide 16 of the lecture notes 13 on undirected graphs (except that, for this project, the graph will be connected). Specifically, the first line of input consists of a single integer V, the number of vertices in the graph. The second line is also a single integer E, the number of edges in the graph. This is followed by E edges, one edge per line, each consisting of two integers u and v, separated by a space. The edges do not appear in any specific order.

The desired output starts with a line consisting of a single integer K, the minimum number of directed edges that would have to be added to the directed graph in order to make it strongly connected (hence if K is 0 then the graph is strongly connected without adding any new directed edges to it). This is followed by a listing (one edge per line) of the resulting directed edges (each as an ordered pair of vertices separated by a space), in the same order in which the depth-first search assigned directions to them. Note that you list the directed edges whether the directed graph is strongly connected or not.

Example

The input file contains: 

	7
	8
	0 5
	4 3
	0 1
	6 4
	5 4
	0 2
	0 6
	5 3
which represents the connected undirected graph below:

The adjacency list of vertex 4 should then contain 3, 5, 6 in that order. The depth-first search would start at vertex 6 and would visit the graph's vertices in the following order: 

	6, 0, 1, 2, 5, 3, 4

The resulting depth-first tree would have the following tree edges (listed as (parent, child) pairs): 

	(6,0), (0,1), (0,2), (0,5), (5,3), (3,4)

It would have the following two non-tree edges (listed as (descendent,ancestor) pairs): 

	(4,5), (4,6)

The order in which the depth-first search assigns the above directions to the edges would be: 

	(6,0), (0,1), (0,2), (0,5), (5,3), (3,4), (4,5), (4,6)

The desired output file for this example would then contain: 

	2
	6 0
	0 1
	0 2
	0 5
	5 3
	3 4
	4 5
	4 6

In the above output, K is 2 because a minimum of two directed edges must be added to the directed graph in order to make it strongly connected. (It is entirely coincidental that the tree edges appeared first. In general you could have some non-tree edges appear before some tree edges.)
