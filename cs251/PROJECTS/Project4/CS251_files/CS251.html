Part 1. Cuckoo Hashing

Overview

Cuckoo hashing is a scheme for resolving hash collisions of values of hash functions in a table, with worst-case constant lookup time.

The name derives from the behavior of some species of cuckoo, where the cuckoo chick pushes the other eggs or young out of the nest when it hatches; analogously, inserting a new key into a cuckoo hashing table may push an older key to a different location in the table.

Algorithm description

The basic algorithm is as follows. We assume here that the data structure consists of a hash table of size r with two hash functions (h1 and h2). When a new (key, value) pair is inserted, the key is first hashed by h1, which yields an index i1 in the table. If the corresponding cell is empty, the key is inserted there. If not, the second hash function h2 is evaluated to produce a second index i2. If that cell is empty, the new key is inserted there, otherwise it is inserted at index i1, thereby dislodging the key previously stored there. When this happens, the dislodged key is moved to its other possible location, which may dislodge yet another key. This process is repeated until an empty cell is found and all keys are in one of their two possible locations. There may be cases, however, where this sequence of replacements forms a loop, in which case no solution exists. To prevent the algorithm from getting stuck in a loop or an excessively long sequence, an upper bound is prescribed on the length of the sequence of replacements. If that length is exceeded, a rehash occurs: the size of the hash table is doubled, and all elements that were previously stored in the table must be re-inserted in the order they were stored in. Note that re-insertion in this manner may cause other elements to be relocated, and may even trigger additional rehashes. Once this rehashing is complete, the key that remained nestless after the last key swap can be inserted into the new table.

Your first task is to create a class Cuckoo.java that implements this algorithm. Specifically, your solution must satisfy the following requirements:

  • Admit Key and Value types as generic types
  • Implement the functions put(Key, Value), get(Key), delete(Key), contains(Key), and size()
  • Use the prescribed family of hashing functions (see below †)
  • Adjust the table size r such that r≥2×(1+ϵ)n at all times, where n is the number of keys stored in the data structure, starting from an initial size r=28=256 and automatically rehashing (doubling r) whenever necessary to maintain the inequality. You will choose ϵ=0.05.
  • Set the upper bound on the length of a sequence to Lmax=⌈3log1+ϵr+1⌉
  • If the length of a sequence of evictions exceeds Lmax, double the size r of the table, and rebuild.
  • Allow for verbose mode to be turned on and off through a function setVerbose(boolean) to control whether the trace of the algorithm is to be printed to standard output or not (see below ‡).

† Hash functions: Central to Cuckoo hashing is the choice of independent pairs of efficient hash functions out of a family whose members achieve a near uniform hashing of the set of possible keys over the interval [0,...,r−1]. For this project you will use hash functions of the form ha(x)=a x216modr, where a is a large prime number and x is a positive integer. Note that changing the size r of the table also affects the hash functions.

‡ verbose mode: Whenever the verbose mode is switched on via setVerbose(true) your algorithm must output its trace to StdOut during the put(Key, Value) procedure. Specifically, upon entering the function, your code must print out every table modification. 

(<ID> <key> <value>) ...
where <ID> is the insertion index, and <key> and <value> form the (key, value) pair inserted. Note that each time a key is being dislodged from its current position in the table it is reinserted at its other possible location, which must be printed out. In the event that the insertion creates a sequence that exceeds the maximum authorized length, your code must print out: 

(hash <a1> <a2> <r>)

to indicate that the contents of the tables will be rehashed using two new hash functions parameterized by the integers a1 and a2, respectively, and the new size of the hash table, r. Then a new sequence of table insertions must be printed out that corresponds to the sequence of insertions performed in place with the new functions, starting from the first valid key present in the table and proceeding until all entries have been repositioned. Only then should you attempt to reinsert the nest-less key that forced a rehash. Note that your verbose output must use the same syntax at the beginning to indicate what hashing parameters are used initially.

Note: Additional details about Cuckoo hashing can be found in the original paper Cuckoo Hashing by Pagh and Rodler.

Implementation

Create a class Cuckoo.java that implements the above data structure. Your code should process input from StdIn with the following format. The first line contains the two large prime numbers a1 and a2, separated by a space. The second line contains a single integer n, the number of commands to process. This is followed by n lines, each specifying a different command. The possible commands, their format, and the expected responses are described below:

  • put <key> <value>
    • Insert the pair (key, value) into the hash table.
    • The output from this command is described above, with verbose mode on.
  • get <key>
    • Print the value corresponding to the key
    • If the key is not in the table, print "none" (without quotes).
  • delete <key>
    • Delete the key from the table.
    • This command does not have any output.
  • contains <key>
    • Print "yes" if the key is in the table, or "no" otherwise (without quotes).
  • size
    • Print the number of elements in the table.

For this assignment, all keys and values are integers. Example input files are named cuckoo*.txt, and example output files are named cuckoo*out.txt.

Part 2. Matching Sums

Overview

We saw in class an N2 logN sorting-based solution to the 3-sum problem. Given a series of numbers a0, a1, ..., an−1 the 3-sum problem consists in finding all triplets (ai, aj, ak) such that ai+aj+ak=0. For this task we consider a closely related version of this problem in which the goal consists in finding all the quadruples (ai, aj, ak, al) with i≠j≠k≠l that satisfy following condition: ai+aj=ak+al.

Tasks description

You will implement two solutions to this problem. Your first solution must use sorting to solve the problem and must achieve a complexity of O(n2logn). Your second solution must use hash tables to solve the problem and have a complexity of O(n2).

Implementation

Create two classes SumSort.java and SumHash.java that implement the two solutions mentioned above. Both programs should process their input using StdIn.java in the following format.

The first line contains a single integer n, the number of values to read. It is followed by a list of n integers a0, a1, ..., an−1, one per line. Both programs must print out their results to the command line in the following format: the first value output is the number of quadruples found followed by all the quadruples in the form "i0 i1 i2 i3" printed on separate lines such that ai0+ai1=ai2+ai3, and i0<i1 and i0<i2<i3. The individual quadruples must be listed in lexicographical order.

We are providing several examples with solutions under values*.txt and match*.txt.

Part 3. Ferry Loading Problem

Overview

We consider the case of a ferry that transports cars across a river. The ferry comprises two lanes of cars throughout its length. Before boarding on the ferry, the cars wait on a single queue in their order of arrival. When boarding, each car can go to either the left or the right lane. Boarding stops when the next car in the queue does not fit in the space left in either lane.

Given the length L∈IN of the ferry and the lengths li∈IN of all cars i≥0 in the queue, what is the maximum number of cars that can be loaded onto the ferry?

  • Note 1: The total number of boarded cars and the length of the right lane (k,len) can be used to characterize an intermediate configuration.
  • Note 2: Given a state (k,len), two possibilities exist for the next car to board: (k+1,len+lk+1) if the car is added to the right lane or (k+1,len) if the car is added to the left lane.

Implementation

Create a class Ferry.java to implement an efficient solution to this problem using a hashing table. Your program should obtain the value of the constant L as an argument, and using StdIn.java import the car information in the following format.

The first line contains the number n of car lengths values to read. It is followed by a list of n integers l0, l1, ..., ln−1, one per line, each corresponding to the length of a car in the queue. Your program must print out to standard output the maximum number of cars that can be loaded on the ferry.

We are providing several examples under cars*.txt. The solution for each file corresponding to different values of L are included in maxcars*.txt.
